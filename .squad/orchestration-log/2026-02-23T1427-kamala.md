|----------|-----------|--------|
| D1: Separate Code/Doc Services | Different Copilot strategies, context needs, failure severity | Riri: implements both interfaces separately |
| D2: TransformationResult per file | Granular success reporting, per-file retry, token accounting | Supports partial PR generation |
| D3: Sequential Copilot calls | Rate limit safety, debuggability | Can add controlled parallelism post-launch |
| D4: Git Data API for multi-commit | No local clone needed, remote-first, precise control | America: builds Git Data API integration |
| D5: Branch recreate on retry | Clean state, no merge commits on automation branches | Idempotency via delete-and-recreate |
| D6: Failed files excluded from PR | Clean separation of validated changes only | Reviewers see only successful transformations |
| D7: UpgradeOrchestrator as pipeline owner | Single testable entry point | Kate: mocks all four services for E2E tests |
